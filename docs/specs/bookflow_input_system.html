Three variants

Text field, search field, textarea.

All three share the same base styling system, the same cream-50 background, the same forest-800 focus state. They differ only in structure: search has a leading icon, textarea is multi-line. Every other input in the app is one of these three.

Text field — the default

Used everywhere a single-line value is needed: name, email, password, search, settings values. Optional leading and trailing adornments.

Search field — leading-icon by convention

Always has a leading search icon. May have a trailing clear button when populated. Used in Library, Discover, and the main app search.

Textarea — for longer-form input

Multi-line. Min height 80px. Resizable vertically by default. Used for feedback messages, notes, and the Q&A input field. Auto-grows as the user types up to a max height of 240px before scrolling internally.

Seven states

Every variation a real input goes through.

An input in production travels through more states than buttons. We define seven that matter: enabled, disabled, focus, pressed, error, success, autofill. Note that focus and pressed look nearly identical — the difference is intent (focus = ready to type, pressed = actively typing).

Icon library

Lucide React Native — configured for Bookflow.

Lucide is the icon library for the entire system: 1,400+ icons, clean editorial line style that matches the design language, tree-shakable so the bundle stays small. We override the default 2px stroke to 1.5px for a slightly lighter feel that matches Fraunces and Geist's character.

Installation & setup

# Install for Expo / React Native npx expo install lucide-react-native npx expo install react-native-svg

Why two packages? Lucide React Native renders SVG natively. react-native-svg provides the SVG rendering engine. Both are required.

Default usage pattern

Import only the icons you use. Tree-shaking ensures unused icons don't bloat the bundle. Always pass strokeWidth={1.5} to match the design language.

// Import individual icons — tree-shakable import { Search, Mail, Lock, Eye } from 'lucide-react-native'; // Use with consistent stroke and color <Search size={18} strokeWidth={1.5} color={tokens.colors.ink[700]} />

Wrap it: build a single <Icon /> component

Don't pass strokeWidth={1.5} manually every time — that's how inconsistencies creep back in. Wrap Lucide once, lock the defaults, use the wrapper everywhere.

// components/Icon.tsx import * as Lucide from 'lucide-react-native'; import { tokens } from '~/design/tokens'; type IconName = keyof typeof Lucide; export function Icon({ name, size = 18, color = tokens.colors.ink[700], }: { name: IconName; size?: number; color?: string; }) { const Component = Lucide[name] as React.ComponentType<any>; return <Component size={size} strokeWidth={1.5} color={color} />; } // Usage anywhere <Icon name="Search" /> <Icon name="Mail" size={14} color={tokens.colors.error} />

Usage rules

Six rules for inputs.

The same kind of guardrails we set for buttons. These prevent the common patterns that fragment input systems over time.

  1. 01
    Always include a label, even when it seems redundant.
    Placeholder is not a label. Screen readers ignore placeholder text. If a visible label feels redundant, hide it visually with a sr-only class but keep it in the DOM.
  2. 02
    Validate on blur, not on every keystroke.
    Showing "invalid email" while the user is mid-typing feels punishing. Wait until they've finished and moved on. The exception: password strength and character counters update live.
  3. 03
    Error messages explain what's wrong AND how to fix it.
    Bad: "Invalid input." Good: "Please enter a valid email address — we'll send a verification link." The fix is implicit but present.
  4. 04
    Success state is for high-stakes confirmations only.
    Don't add a green checkmark every time someone types something correctly — that's noise. Reserve success state for verification (email verified, password strength sufficient, .edu confirmed).
  5. 05
    Disabled inputs need a "why" explanation.
    A field that's grayed out without explanation feels broken. Always pair with helper text: "Email can't be changed once verified" or "Edit your name in Account settings."
  6. 06
    Use the right input type for keyboard hints on mobile.
    Email → type="email", phone → type="tel", search → type="search". Wrong type means wrong on-screen keyboard — frustrating on mobile.
